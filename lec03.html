Лекция 3. Лейбницевское равенство и равенство по определению. Арифметика Пеано

Лейбницевское равенство

Теория называется теорией с равенством, если она содержит (универсальные замыкания) следующих аксиом
  • (E1) x = x
  • (E2) x = y → y = x
  • (E3) x = y → y = z → x = z
  • (E4) x = y → f x = f y и аналогично для многоместных функциональных симовлов
  • (E5) x = y → P x → P y и аналогично для многоместных предикатных символов
Напоминание: A[t/x] обозначает результат подстановки терма t вместо свободных вхождений переменной x в формулу A.
Утверждение. Пусть A есть произвольная формула, x ∈ FV(A), s, t — термы. Тогда |- s = t → A[s/x] → A[t/x].
Доказывательство проходит индукцией по формуле A.
Можно взять это утверждение и рефлексивность как аксиомы равенства. Симметричность и транзитивность выводятся (упражнение). Такое равенство называется лейбницевским: равными считаются объекты, взаимозаменяемые в любом контексте.

Тактика reflexivity


--------
 t = t

Тактика rewrite


A[s/x] s = t
--------------
    A[t/x]
Если есть посылка H : s = t, то rewrite <- H превращает цель A[t/x] в A[s/x], а rewrite H превращает цель A[s/x] в A[t/x]. Допустимо также rewrite H1, H2, H3 для последовательных переписываний.

Theorem eq_refl : ∀ x : nat, x = x.
Proof.
intro x. reflexivity.

intro не обязательно, так как reflexivity делает intro, пока не доберется до равенства
Qed.

Theorem eq_symm : ∀ x y : nat, x = y → y = x.
Proof.
intros x y H.
rewrite <- H. reflexivity.
Qed.

Theorem eq_trans : ∀ x y z : nat, x = y → y = z → x = z.
Proof.
intros x y z H1 H2.

now tactic эквивалентно tactic; easy где easy — тактика, доказывающая простые цели. Другие автоматические тактики: trivial, auto.

now rewrite H1, H2.
Qed.

Кроме тактик, которые начинаются с маленькой буквы, в Coq имеются команды, начинающиеся с большой буквы. Команды печатают диагностику или меняют различные режимы работы Coq, например, количество печатаемой информации. Команды не изменяют состояние доказательства. В CoqIDE тактики находятся в меню Try Tactics, а некоторые полезные команды — в меню Queries.
Команда Print печатает информацию об идентификаторе: определении, типе и т.д. Она не приманяется к составным выражениям.

Print nat.

Команда Check проверяет любое выражение на синтаксическую и тИповую правильность.

Check S (S (S O)).

Рекурсивные определения начинаются с команды Fixpoint.

Fixpoint plus (m n : nat) : nat :=
match m with
| 0 ⇒ n
| S m' ⇒ S (plus m' n)
end.

Смысл конструкции match следующий: если m = 0, то значение выражежния есть 0. Если же m имеет вид S m' для некоторого m', то возвращается S (plus m' n).
Определим инфиксный оператор +, являющийся записью функции plus.

Infix "+" := plus (at level 50, left associativity) : nat_scope.

Fixpoint times (m n : nat) : nat :=
match m with
| 0 ⇒ 0
| S m' ⇒ n + times m' n
end.

Infix "*" := times (at level 40, left associativity) : nat_scope.

Можно использовать Coq как калькулятор

Compute 10 + 20.
Compute 2 * 3 * 4 * 5.

Равенство по определению

Coq имеет два вида равенства: равенство по определению и доказуемое (лейбницевское) равенство. Термы, равные по определению, считаются одним и тем же термом.
Определение функции plus выше добавляет следующие равенства по определению:
  • (P1) 0 + n = n
  • (P2) S m + n = S (m + n)
Определение функции times добавляет
  • (T1) 0 * n = 0
  • (T2) (S m) * n = n + m * n
Это позволяет заменить некоторые доказательства вычислениями. Так, если использовать только лейбницевское равенство, то для утверждения 2 + 2 = 4, то есть S (S 0) + S (S 0) = S (S (S (S 0))), можно привести следующее довольно длинное доказательство.
  • 1. S (S 0) + S (S 0) = S (S 0 + S (S 0)) (P2)
  • 2. S 0 + S (S 0) = S (0 + S (S 0)) (P2)
  • 3. 0 + S (S 0) = S (S 0) (P1)
  • 4. 0 + S (S 0) = S (S 0) → S (0 + S (S 0)) = S (S (S 0)) (E4)
  • 5. S (0 + S (S 0)) = S (S (S 0)) 3, 4, Modus Ponens (MP)
  • 6. S 0 + S (S 0) = S (0 + S (S 0)) → S (0 + S (S 0)) = S (S (S 0)) → S 0 + S (S 0) = S (S (S 0)) (E3)
  • 7. S 0 + S (S 0) = S (S (S 0)) 2, 5, 6, MP
  • 8. S 0 + S (S 0) = S (S (S 0)) → S (S 0 + S (S 0)) = S (S (S (S 0))) (E4)
  • 9. S (S 0 + S (S 0)) = S (S (S (S 0))) 7, 8, MP
  • 10. S (S 0) + S (S 0) = S (S 0 + S (S 0)) → S (S 0 + S (S 0)) = S (S (S (S 0))) → S (S 0) + S (S 0) = S (S (S (S 0))) (E3)
  • 11. S (S 0) + S (S 0) = S (S (S (S 0))) 1, 9, 10, MP
Если же считать обе части равенств (P1) и (P2) одним и тем же термом, то S (S 0) + S (S 0) и S (S (S (S 0))) также будут одинаковыми термами, и их равенство доказывается одной аксиомой (E1) (рефлексивность).
Различие между этими подходами описывается так называемым принципом Пуанкаре: «Вычисление и доказательство — не одно и то же. Вычисление не требует доказательства». Coq использует принцип Пуанкаре, и это позволяет значительно сократить размер доказательств, а также делает возможным написание тактик, где основная работа заключается в вычислении функций, написанных на самом языке Coq.
Как бинарное отношение равенство по определению содержится в лейбницевском (доказуемом) равенстве. Отличие заключается в том, что равенство по определению разрешимо, в то время как доказуемое равенство нет. Поэтому Coq может автоматически проверить истинность S (S 0)) + n = S (S n), поскольку это равенство следует из (P1) и (P2). Однако равенство n + S (S 0) = S (S n) нужно доказывать, так как с точки зрения Coq части этого равенства — разные термы.

Theorem plus_0_l : ∀ n, plus 0 n = n.
Proof.
intro n. reflexivity.

Равенство по определению

Qed.

Theorem plus_S_l : ∀ m n, plus (S m) n = S (plus m n).
Proof.
intros m n. reflexivity.
Qed.

Lemma plus_0_r : ∀ n, plus n 0 = n.
Proof.

reflexivity не работает: это не есть равенство по определению

induction n as [| n IH].

Так мы даем имена переменной и индукционной гипотезе в шаге индукции. Можно просто induction n; тогда Coq выберет свои имена. База индукции
* reflexivity.
Шаг индукции. Тактика simpl переписывает слева направо имеющиеся равенства по определению, в данном случае (P2)
* simpl. rewrite IH. reflexivity.
Qed.

Lemma plus_S_r : ∀ m n, m + S n = S (m + n).
Proof.
induction m as [| m IH].
* intro n. reflexivity.
* intro n. simpl. now rewrite IH.
Qed.

Theorem plus_comm : ∀ m n, m + n = n + m.
Proof.
induction m as [| m IH].
* intro n. rewrite plus_0_r. reflexivity.
* intro n. simpl. rewrite plus_S_r, IH. reflexivity.
Qed.

При переписывании с помощью тактики rewrite H может оказаться, что в цели есть несколько подвыражений, имеющих форму левой (или правой) части равенства H. Здесь H может быть гипотезой в контексте (сверху от горизонтальной черты) или теоремой, доказанной ранее. Например, пусть цель есть
S (n + m + m * n) = S (m + n + m * n)
(напоминание: + есть левоассоциативная операция), и мы хотим заменить n + m на m + n с помощью доказанной выше теоремы plus_comm. В цели есть несколько подтермов, имеющих вид x + y:
  • n + m
  • (n + m) + m * n
  • m + n
  • (m + n) + m * n
Указать, какой из этих подтермов нужно переписывать, можно так:
  • rewrite (plus_comm n m) переписывает n + m в m + n
  • rewrite (plus_comm (n + m) (m * n)) переписывает (n + m) + m * n в m * n + (n + m).
